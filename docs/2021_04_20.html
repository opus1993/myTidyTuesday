Feature Engineering

Given the description, cast members, director, and release_year, can we build a useful model to predict the genre?

Unbalanced multi-class problems are especially challenging. And in the Netflix dataset, most movies belong to two or three genres at the same time. To make this problem just a little less complex, we will bundle some genre’s into logical groupings. I am also only using the first listed_in genre, as the primary. The result is 20 genres.

netflix_df <- netflix_titles %>%
  select(
    genre = listed_in,
    show_id,
    description,
    cast,
    director,
    release_year
  ) %>%
  separate_rows(genre, sep = ", ") %>%
  mutate(genre = case_when(
    str_detect(genre, "International") ~ "International",
    str_detect(genre, "Romantic") ~ "Romantic",
    str_detect(genre, "Drama") ~ "Drama",

Let’s split the data using the default 3:1 ratio of training-to-test and resample the training set using 10-fold cross-validation. V-fold cross-validation randomly splits the data into V groups (10) of roughly equal size (called “folds”). A resample of the analysis data consists of V-1 (9) of the folds while the assessment set contains the final fold. In basic V-fold cross-validation (i.e. no repeats), the number of resamples is equal to V.

The strata argument causes the random sampling to be conducted within the stratification variable.

Our dataset will be pre-processed through two different recipes that engineer text features by tokenizing the description, cast, and the director fields.

The first recipe applies filters to use the words, cast members, and director with the 30 highest term frequencies after removing stop words and stemming.

The second filter the description words to remove stopwords and then builds 25 word embeddings with Glove Twitter 27B. Learn more about word embeddings and pre-trained model transfer learning here. The recipe goes on to hash the cast member and director features into numeric variables with the hashing trick.

Both models are centered and scaled. There are no missing values. With a big multi-class challenge, we need to take action to balance the classes. downsample discards members of the majority classes (in the training set only).

smote_rec <- recipe(genre1 ~ description + cast + director, data = netflix_train) %>%
  step_tokenize(description) %>%
  step_stopwords(description) %>%
  step_stem(description) %>%
  step_tokenfilter(description, max_tokens = 30) %>%
  step_tokenize(cast,
    token = "regex",
    options = list(pattern = ", ")
  ) %>%
  step_tokenfilter(cast, max_tokens = 30) %>%
  step_tokenize(director,
    token = "regex",
    options = list(pattern = ", ")
  ) %>%
  step_tokenfilter(director, max_tokens = 30) %>%
  step_tf(description) %>%
  step_tf(cast) %>%
  step_tf(director) %>%
  step_smote(genre1) %>%
  step_normalize(all_numeric())

embeddings_rec <- recipe(genre1 ~ description + cast + director, data = netflix_train) %>%
  step_tokenize(description) %>%
  step_stopwords(description) %>%
  textrecipes::step_word_embeddings(description,
    embeddings = textdata::embedding_glove27b()
  ) %>%
  step_tokenize(cast,
    token = "regex",
    options = list(pattern = ", ")
  ) %>%
  step_texthash(cast, num_terms = 50) %>%
  step_tokenize(director,
    token = "regex",
    options = list(pattern = ", ")
  ) %>%
  step_texthash(director, num_terms = 50) %>%
  step_smote(genre1) %>%
  step_normalize(all_numeric())

For the models themselves, we use the the parsnip package to create sets of model specifications. All are setup for classification. For this demonstration, I am interested in the behavior of the SVM Liblinear engine and the naive Bayes engine.

ranger_spec <-

  set_engine("naivebayes")

Workflowsets

This is where the cool part comes in: We now define a “workflow set.” We apply cross = TRUE to look at all combinations of recipes and models.

(wfs <-
  bind_rows(workflow_set(
    preproc = list(
      smote = smote_rec,
      embeddings = embeddings_rec
    ),
    models = list(nb_spec, ranger_spec),
    cross = TRUE
  ))
)

And we see we have our four different workflows set up.
